# ქართული STT-ის პოსტ-კორექცია (ლოკალური)



პროექტი ქართულ Speech-To-Text-ს მინიმალურად და უსაფრთხოდ აკორექტირებს.  
ჯერ სისტემას აქვს ლექსიკონზე და მარტივ კონფუზიის/ჰეურისტიკებზე დაფუძნებული „სწრაფი“ შესწორება, შემდეგ (საჭიროების შემთხვევაში) ლოკალურ `Ollama`-ს უტარდება ძალიან მკაცრი პრომპტი.
თუ მოდელის გამომავალი ინპუტისგან „ძალიან შორს“ არის ან არსებობს მორფოლოგიური ბუნდოვანება აუდიოს გარეშე, სისტემა ინარჩუნებს ორიგინალს და არ „იგონებს“.

## რა გჭირდებათ

1. **Windows 11** და **Python 3.10+**
2. **[Ollama for Windows](https://ollama.com/download)** — უფასო, ლოკალურად მუშაობს
3. RAM: სხვა აპები დახურეთ. რეალისტური ნაგულისხმევი არის **1.5B–3B** ზომის მოდელი (დაახლოებით ~12GB ჯამურ RAM-ზე).

## Ollama-ის დაყენება

ინსტალაციის შემდეგ PowerShell-ში:

```powershell
ollama pull qwen2.5:1.5b
```

თუ ეს თეგი არ გაქვთ, სცადეთ:

```powershell
ollama pull qwen2.5:3b
```

ან სხვა პატარა მრავალენოვან ჩატ-მოდელს. შეგიძლიათ მკაფიოდ მიუთითოთ:

```powershell
$env:OLLAMA_MODEL="qwen2.5:1.5b"
```

## ქართული ლექსიკონი (`ganmarteba.ge`-დან)

სისტემას შეუძლია გამოიყენოს **ოფლაინ სიტყვების სია**: `ganmarteba.ge`-ის საჯარო ძიებიდან მიღებული სიტყვები უკვე არის data\ganmarteba_words.txt-ში

## გაშვება

ნიმუშზე:

```powershell
geostt-correct examples\sample_stt.txt -o out.txt --verbose
```

`stdin/stdout`:

```powershell
Get-Content examples\sample_stt.txt -Raw | geostt-correct - --verbose
```

Debug ანგარიში (ჩანაწერის/სეგმენტის დონეზე `skip/reject/ok`):

```powershell
geostt-correct examples\sample_stt.txt --json-report report.json --verbose
```

## API რეჟიმი (ინტეგრაციისთვის)

სერვერის გაშვება:

```powershell
geostt-api --host 0.0.0.0 --port 8000
```

endpoint-ები:

- `POST /v1/correct/prellm` - შედეგი LLM-მდე (heuristics + lexicon)
- `POST /v1/correct/final` - სრული შედეგი (LLM ჩართული თუ `GEOSTT_OLLAMA=1`)

request მაგალითი:

```json
{"text":"გუშინ მე და ჩემი მეგობარი წავედით პარკში"}
```

## ლექსიკონის/მოდელის ჩართვა-გამორთვა

თუ გინდათ „მხოლოდ ლექსიკონით“ სწრაფი და განმეორებადი შედეგი, LLM გამორთეთ:

```powershell
$env:GEOSTT_OLLAMA=0
```

| ----------------------- | ---------------------------------------------------- | --------------------------------------------------------------- |
| `OLLAMA_HOST`           | `http://127.0.0.1:11434`                             | Ollama API მისამართი                                            |
| `OLLAMA_MODEL`          | `qwen2.5:1.5b`                                       | მოდელის სახელი                                                  |
| `OLLAMA_TEMPERATURE`    | `0.1`                                                | დაბალი = უფრო კონსერვატიული                                     |
| `MAX_CHUNK_CHARS`       | `800`                                                | მაქს სიმბოლოები ერთ მოთხოვნაში                                  |
| `MIN_SEQUENCE_RATIO`    | `0.55`                                               | Reject თუ გამომავალი ძალიან განსხვავდება ინპუტისგან             |
| `MAX_RELATIVE_LENGTH`   | `1.55`                                               | Reject თუ გამომავალი მკვეთრად გრძელია                           |
| `GEOSTT_LEXICON`        | *(default: `data/ganmarteba_words.txt` თუ არსებობს)* | ლექსიკონის ფაილის გზა (ერთი სიტყვა/ხაზი)                        |
| `GEOSTT_LEXICON_ENABLE` | `1`                                                  | `0` ნიშნავს ლექსიკონის გადაფასების გამორთვას                    |
| `GEOSTT_OLLAMA`         | `1`                                                  | `0` ნიშნავს LLM-ის გამორთვას (მხოლოდ ლექსიკონი)                 |
| `GEOSTT_HEURISTICS`     | `1`                                                  | `0` ნიშნავს `heuristics.py`-ში არსებული ხელით წესების გამორთვას |


## პაიპლაინი

პაიპლაინი მუშაობს ასე:
`heuristics.py` (whole-token შესწორებები) → `lexicon.py` (OOV-ის გადამოწმება `ganmarteba_words.txt`-ით, phonetic variants-ის გამოყენებით) → (სურვილისამებრ) `Ollama` მკაცრი პრომპტით → `safety.py` საბოლოო მიღება/უარყოფა (რომ ზედმეტად არ გადახტეს ინპუტიდან).

## უსაფრთხოება და ბუნდოვანება

ქართულ STT-ში შესწორება ყოველთვის ერთმნიშვნელოვანი არ არის.  
მაგალითად,  **ბავშვს** და ასევე **ბავშვებს** — ორივე შეიძლება გრამატიკულად ჩაითვალოს კონტექსტის გარეშე.

ამიტომ:

1. LLM-ის პრომპტი ავალებს დატოვოს უკვე სწორი მორფოლოგია.
2. `safety.py` ამოწმებს მსგავსებას/სიგრძეს და „გადახტომებს“ reject-ს უკეთებს.
3. `safety.py`-ში არსებობს „აუდიოს გარეშე ექვივალენტური“ რამდენიმე წყვილი; ასეთ დროს ერთ-ტოკენიანი swap reject-დება და ინპუტი რჩება.

ასევე, ლექსიკონის ეტაპზე დაცულია პრაქტიკული წესები:

1. `heuristics.py`-ით წინასწარ დაფიქსირებულ ტოკენებს ლექსიკონი არ უნდა გადააწეროს.
2. ძალიან მოკლე ტოკენებზე ლექსიკონის მიხედვით ერთნიშნიანი „ფლიპები“ უფრო იშვიათად კეთდება, რადგან ეს ხშირად ASR-ის ოსცილაციის ბუნებრივ შედეგს ასახავს.
